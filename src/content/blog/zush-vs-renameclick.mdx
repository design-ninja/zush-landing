![Zush AI rename results showing files with descriptive before and after names and green checkmarks](/images/showcase/macos/batch-rename-light.webp)

Zush is a desktop AI file renamer for mixed folders. The current product surface is broad: 100+ formats across images, RAW photos, design files, documents, iWork and Office files, subtitles, videos, and audio. It also has templates, 145+ Naming Blocks, custom prompts, folder monitoring, full rename history, Finder tags, Spotlight-friendly metadata, BYOK, and Offline AI via Ollama.

The important part in this comparison is that Zush does not force a file-type silo. A folder with `IMG_4821.HEIC`, `_DSC0019.NEF`, `checkout-flow.fig`, `voice-memo.m4a`, `client-brief.pages`, `recording.mov`, and `invoice.pdf` can stay in one workflow.

Pricing is simple: $8/month or $38 one-time for PRO. Both paid plans unlock unlimited PRO renames, BYOK, and Offline AI. Templates, Naming Blocks, and custom prompts are included free.

## RenameClick

![RenameClick desktop app interface (image: rename.click)](/images/blog/competitors/renameclick.webp)

RenameClick is a local-first desktop app for macOS and Windows. Its core pitch is privacy: after the one-time local model download, its built-in Qwen3-VL provider can process files on the device without an API key or internet connection. It also supports LM Studio and Ollama for local server workflows, plus OpenAI, Google Gemini, and Alibaba Cloud Qwen when you explicitly connect cloud API keys. RenameClick notes that cloud providers are not available in the Mac App Store version; use the direct download if those providers matter.

RenameClick is not only a renamer. It is closer to a file organizer: it can rename files, categorize them, move them into folders, run Auto Flow watch rules, search by content with AI Search, and build names from EXIF/date placeholders. Its docs list supported images and documents only: common images including HEIC/HEIF/TIFF/SVG and documents including PDF, DOC, DOCX, TXT, MD, CSV, RTF, and ODT. The docs explicitly list video, audio, archives, executables, and app bundles as unsupported.

## Format support

### Zush is broader for real mixed folders

Zush supports RAW photo formats, design files, audio, videos, iWork files, Office files, subtitles, and data/text formats. That is the practical advantage if your folders are messy because they are real: screenshots beside phone photos, camera RAWs beside PDFs, Figma/Sketch/PSD files beside Keynote/Pages documents, and video/audio files beside meeting notes.

### RenameClick covers common images and documents

RenameClick covers common image and document workflows well: JPG/JPEG, PNG, GIF, WebP, BMP, SVG, HEIC, HEIF, TIFF/TIF, PDF, DOC/DOCX, TXT, Markdown, CSV, RTF, and ODT. It also has useful document processing details: text extraction for text PDFs and Office documents, vision fallback for scanned PDFs, and configurable document text trim limits.

The gap is around specialized and media-heavy formats. RenameClick's docs do not list camera RAW formats, video files, audio files, archives, or design files like Sketch/Figma/PSD as supported. If those formats matter, Zush is safer.

## Offline AI and providers

RenameClick wins if you want local-first behavior immediately. Its local provider is the default, the model is bundled as part of the product flow, and the docs describe offline use after the model download with no API key and no API cost. It also exposes advanced Local AI settings like context size, temperature, prompt length, text trim, and batch file limits.

Zush wins if you care more about cloud-provider choice and high-volume BYOK. Zush supports Gemini, Groq, OpenAI, and Claude through BYOK, with keys stored locally, and it supports Offline AI via Ollama when local processing is the priority. Groq and Claude are notable Zush-side options; built-in Qwen3-VL, Alibaba Cloud Qwen, and LM Studio are RenameClick-side options.

The trade-off is simple:

- RenameClick: local AI is the default product identity.
- Zush: BYOK and broad format coverage are the default product identity, with Ollama available for offline workflows.

## Automation and organization

Both tools can watch folders, but they emphasize different outcomes.

Zush folder monitoring is built around automatic AI renaming with reusable templates and Naming Blocks. It is a good fit for Downloads cleanup, screenshots, recurring client folders, and predictable naming conventions where new files should get searchable names as they arrive.

RenameClick Auto Flow is more of a rule-row system. Each row can have its own source folder, destination folder, naming format, custom instructions, language, sorting preset, recursive setting, and Auto Apply setting. It can run multiple rows at the same time, wait for files to become stable before processing, avoid looped processing inside destination folders, and pause when free-plan credits run out.

If your automation goal is "rename files consistently", Zush is cleaner. If your automation goal is "rename and route files into folder structures", RenameClick is stronger.

## Search, metadata, and retrieval

Zush helps retrieval by generating meaningful filenames, adding Finder tags, and writing Spotlight-friendly metadata. That fits Mac users who already search through Finder/Spotlight and want filenames to carry the most important context.

RenameClick adds a separate AI Search workspace. It lets you search by content meaning, not just filename keywords. For example, a search for invoices can match PDF scans even if the original filename is `scan_001.pdf`. AI Search is read-only for rename/move purposes, but it can reveal files, exclude results, or move items to trash.

For discovery inside chaotic folders, RenameClick's AI Search is a meaningful differentiator. For turning files into long-term searchable assets across the OS, Zush's filename + tag + metadata workflow is the better everyday layer.

## Migration tip

If you are coming from RenameClick to Zush, test the files that RenameClick does not publicly cover first: RAW photos, design files, audio, videos, and iWork documents. If those are the reason you are switching, the difference will be obvious quickly. If you already use a cloud provider key, add it in Zush BYOK and keep provider costs separate from the desktop app.

If you are coming from Zush to RenameClick, start with the free plan and test the local model on a representative folder. Pay attention to hardware: RenameClick recommends 16 GB system RAM and uses a multi-GB local model. If the built-in model is too generic for your files, try LM Studio, Ollama, or a cloud provider before judging the workflow.

## FAQ

### Is RenameClick cheaper than Zush?

No. Zush is $8/month or $38 one-time. RenameClick is $8+ VAT/month or $48+ VAT lifetime. The difference is small on monthly plans, but Zush is $10 cheaper on lifetime before tax.

### Does RenameClick work offline?

Yes. RenameClick is local-first. After the local model is downloaded, the built-in local provider can run without internet, without an API key, and without API costs. Cloud providers require internet and your own provider key.

### Does Zush work offline?

Yes, through Offline AI mode via Ollama. The difference is that Zush does not bundle its own local model as the default path; you set up Ollama and choose a compatible local model.

### Does RenameClick support RAW photos?

RenameClick's public supported-file list does not include camera RAW formats like CR2, CR3, NEF, ARW, DNG, ORF, RAF, RW2, PEF, SRW, or SR2. Zush explicitly supports those RAW formats.

### Does RenameClick support video or audio files?

No, not according to its current docs. RenameClick lists common image and document formats and explicitly calls video, audio, archives, executables, and app bundles unsupported. Zush supports common video and audio formats.

### Which has better folder automation?

It depends on the goal. Zush is better for automatic naming with templates and Naming Blocks. RenameClick is better for folder routing because Auto Flow can rename, categorize, and move files into destination folders.
