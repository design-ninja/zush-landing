Templates save a rename setup so you can reuse it later. They are useful when the same kind of folder arrives every week and needs the same naming structure. Start by running a [batch rename](/docs/batch-rename-files/) with the rules you want, then save that setup as a Template.

## What a Template can capture

A Template can combine:

- naming style and format
- language and date format
- [Naming Blocks](/docs/naming-blocks/)
- [custom rename prompt](/docs/custom-prompts/)
- metadata and tag preferences
- file-type behavior for monitored folders
- [AI mode](/docs/ai-modes/) preferences for the workflow

## Good Template examples

| Template | Useful for |
| --- | --- |
| Client deliverables | PDFs, docs, screenshots, design files, video exports |
| Expense receipts | PDFs, scans, screenshots, emailed receipts |
| Podcast or audio | WAV, MP3, M4A, transcripts, show notes |
| Screenshots | Bug reports, UI references, research captures |
| Photo imports | RAW, HEIC, JPG, campaign or shoot folders |

## Templates and monitoring

Templates pair well with [folder monitoring](/docs/folder-monitoring/). A Downloads folder might need general cleanup, while a podcast folder needs guest and topic context. Each monitored folder can follow the naming setup that fits it.
