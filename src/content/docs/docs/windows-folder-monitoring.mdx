Folder monitoring lets Zush watch selected Windows folders and rename new supported files as they arrive. It is useful when the same folders keep filling up with vague names like `download.pdf`, `screenshot.png`, `final-final.mp4`, or `IMG_2048.jpg`.

## Recommended folders

- `Downloads` for PDFs, invoices, images, archives, and exports.
- `Pictures\Screenshots` for daily screenshots and screen captures.
- OneDrive folders for synced work documents.
- Project export folders for design files, audio, video, and client deliverables.

## Set up a monitored folder

<Steps>
1. Open Zush and go to the monitoring workflow.
2. Add a Windows folder such as Downloads or Screenshots.
3. Choose which file types should be included or excluded.
4. Pick the naming template or prompt that should apply to new files.
5. Keep review and undo enabled for safer automation.
</Steps>

## Best practices

Start with one noisy folder before monitoring several locations. Downloads and Screenshots usually give the fastest payoff because they collect unrelated files with weak names.

Use exclusions for temporary files, application caches, and formats you do not want Zush to touch. For team folders, confirm the naming convention before applying automated rename workflows.

<Aside type="caution">
Folder monitoring is a naming workflow, not a folder-moving workflow. Zush focuses on descriptive filenames, metadata, review, and undo.
</Aside>
