Custom prompts let you tell Zush how a workflow should name files. Use them when the default naming style is close, but your folder needs a specific convention. For repeatable work, combine prompts with [Templates](/docs/templates).

## Good prompt examples

```text
Keep names short, put the subject first, and avoid generic words like final or screenshot.
```

```text
For receipts, include vendor, date, amount, and document type when visible.
```

```text
For design exports, include product area, screen name, state, and asset type.
```

## Prompt tips

- Give a rule the AI can follow.
- Name the file type or workflow.
- Say what to include and what to avoid.
- Keep prompts reusable if they will become a [Template](/docs/templates).
- Do not include sensitive data unless it is needed for the filename; review the [privacy guide](/docs/privacy-security) first for sensitive folders.

## Rename prompts and tag prompts

Rename prompts control the filename. Tagging prompts guide [tags and metadata](/docs/metadata-tags) where supported. Keep them aligned: if a project, client, or document type matters in one place, it usually matters in the other.
