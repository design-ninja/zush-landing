Zush keeps rename history so Windows users can review previous batches and undo changes. This matters when you are applying AI-generated names to many files at once.

## What history records

For each rename action, Zush stores the original filename, the new filename, the folder, and the timestamp. This lets you inspect what changed and revert the action when needed.

## Undo a batch

<Steps>
1. Open the activity or history view in Zush.
2. Select the batch you want to inspect.
3. Review the original and new filenames.
4. Use revert to restore the previous filenames.
</Steps>

## Safer batch rename workflow

Review suggestions before applying a batch, especially in shared folders or client folders. Regenerate weak names individually instead of rerunning the full batch. For automated folder monitoring, start with one folder and inspect history after the first few runs.

<Aside type="tip">
Keep the first run small. A 20-file folder is enough to validate the naming template before you apply it to hundreds of files.
</Aside>
