## Compare AI modes

| Mode | Best for | Notes |
| --- | --- | --- |
| Cloud AI | Fast setup and strong general results | Uses Zush-managed cloud analysis. |
| BYOK | High-volume cloud use with your provider account | Supports Gemini, Groq, OpenAI, and Claude for PRO users. [Set up BYOK](/docs/byok). |
| Offline AI | Private local processing for supported files | Uses local Ollama models and depends on your hardware and model choice. [Set up Offline AI](/docs/offline-ai). |

## How to choose

Use Cloud AI when you want the simplest path. Use [BYOK](/docs/byok) when you already have a provider account or want unlimited cloud renames through your own key. Use [Offline AI](/docs/offline-ai) when supported local analysis matters more than speed or broadest model quality.

## Quality trade-offs

Cloud models are usually stronger across complex mixed folders. Local models can be useful for private visual and preview-based workflows, but quality depends on the model, file type, and available hardware.
