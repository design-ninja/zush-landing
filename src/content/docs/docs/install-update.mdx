Zush runs on Mac and Windows. Install from the product page for your operating system, then use the same core AI file renaming workflow on either platform.

Quick links: [download Zush for Mac](/download/mac), open [Zush on the Mac App Store](https://apps.apple.com/app/zush/id6758432449), or install [Zush from the Microsoft Store](https://apps.microsoft.com/detail/9ND4WVZSDQ3X).

## Install on Mac

Mac users can install Zush from the [Mac product page](/mac), the [direct signed .dmg download](/download/mac), or the [Mac App Store listing](https://apps.apple.com/app/zush/id6758432449). Depending on the build you choose, updates are handled by the store channel or the signed direct app update flow.

Zush for Mac supports macOS 15.0 and newer. Folder access is granted by the user when files, folders, or monitored folders are selected.

## Install on Windows

Windows users install Zush from the [Microsoft Store listing](https://apps.microsoft.com/detail/9ND4WVZSDQ3X) or the [Windows product page](/windows). The Store handles package trust, installation, and updates through the standard Windows app channel.

Zush for Windows supports Windows 10 and Windows 11.

## Requirements

| Requirement | Notes |
| --- | --- |
| Memory | 4 GB minimum, 8 GB recommended for Offline AI. |
| Disk space | About 200 to 250 MB for the app, plus additional space for local Ollama models. |
| Network | Required for managed Cloud AI and BYOK. Optional for Offline AI after local setup. |
| Local AI | Requires [Ollama](https://ollama.com/download) and a compatible local model. See the [Offline AI setup guide](/docs/offline-ai/). |

## Update behavior

Store builds update through the platform store. Direct builds use the app's normal signed update path. For the best result, keep Zush current before testing new file types, [AI modes](/docs/ai-modes/), or [monitored folder workflows](/docs/folder-monitoring/).
