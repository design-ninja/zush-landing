Start with a small folder. A 10 to 20 file batch is enough to see how Zush reads content, suggests filenames, and lets you edit before anything is renamed.

## First rename batch

<Steps>
1. Install Zush: use the [Mac download](/download/mac), [Mac App Store](https://apps.apple.com/app/zush/id6758432449), or [Microsoft Store](https://apps.microsoft.com/detail/9ND4WVZSDQ3X).
2. Add files or a folder from Finder, File Explorer, Downloads, Desktop, or a project folder.
3. Choose the naming style: a simple AI title, a saved [Template](/docs/templates/), or a pattern built with [Naming Blocks](/docs/naming-blocks/).
4. Review the suggested names. Edit weak names inline or regenerate a single file.
5. Apply the batch when the preview looks right.
6. Open [rename history](/docs/undo-history/) if you need to inspect or undo the batch later.
</Steps>

## What to try first

| Folder | Why it is a good first test |
| --- | --- |
| Downloads | Mixed PDFs, screenshots, images, archives, and exports show how Zush handles variety. |
| Screenshots | Visual content quickly shows whether AI-generated names are useful. |
| Client or project exports | Repeated context helps you test Templates and Naming Blocks. |
| Photos or screen recordings | Media files show the value of content-aware names. |

## Mac and Windows parity

The core workflow is the same on both platforms: add files, review suggested names, apply, [monitor folders](/docs/folder-monitoring/), and [undo](/docs/undo-history/) if needed. Platform differences are mostly about installation and the files app you start from: Finder on Mac, File Explorer on Windows.

<Aside type="tip">
Keep the first run non-destructive by using files you can easily restore. Zush has undo history, but a small test folder makes it easier to evaluate the naming style.
</Aside>
